# Service Scope: Spreadsheet Cleanup Automation

## Purpose

This document explains what this demo service includes, what it does not include, and which assumptions must be confirmed before adapting it to a real client project.

## Included

This type of project may include:

- CSV cleanup;
- basic spreadsheet normalization;
- duplicate detection;
- required-field validation;
- invalid-record separation;
- clean CSV export;
- simple Markdown report;
- execution instructions;
- documented limitations.

## Not Included

This service does not include by default:

- full CRM development;
- SaaS development;
- dashboard development;
- data enrichment;
- email automation;
- lead scoring;
- production database integration;
- long-term maintenance;
- handling real client data inside a public repository.

## Required Before Starting

Before starting a real project, confirm:

- input file format;
- required columns;
- duplicate criteria;
- validation rules;
- expected output format;
- example input;
- example expected output;
- who validates the result.

## Data Safety

For public demos and initial validation, use fictitious, anonymized, or masked data.

Do not commit real client data, credentials, tokens, or sensitive business information to the repository.

## Acceptance Criteria

A project like this can be considered delivered when:

- the input file is processed;
- valid records are exported;
- invalid records are separated;
- duplicates are handled according to the agreed rule;
- the report explains what happened;
- the client can reproduce the execution;
- limitations are documented.
